# Design Context — My Agent

**Product**: internal AI coding-assistant chat tool for engineers at 信川机械工业
(Xinchuan Machinery). Used to browse/search their own MES/OMS/APS/WMS Java+Vue
codebase, confirm bugs, walk through code, and draft GitHub issues.

**Audience**: internal engineers (not consumers). Chinese-primary UI copy,
mixed with English technical terms/identifiers. Users paste real screenshots
of their own product's UI and ask the agent to investigate the underlying code.

**Tone**: precise, technical, "control room for your codebase" — closer to a
git diff viewer / terminal than a friendly consumer chatbot. Confident and
quiet, not playful.

**Existing design language (do not discard — evolve it)**:
- Status-color vocabulary shared across tool calls, issue drafts, and
  permission badges: `moss` = confirmed/success, `amber` = pending, `rust` =
  danger, `wire` = read-only/passive. Defined as CSS custom properties in
  `web/style.css` `:root`.
- Type pairing: JetBrains Mono (display/technical), Public Sans (body),
  Source Serif 4 italic (editorial/human moments — welcome copy, headlines).
- Dark-only theme by deliberate choice (no light-mode toggle) — this is a
  focused internal tool, not a general product needing theme parity.

**2026-07-08 redesign**: neutrals were re-tinted toward the moss hue (were
flat blue-gray, read as generic dark-mode) and given wider tonal steps
(`--canvas` for the message stream vs `--ink-950/900/850/800` for chrome).
Sidebar split into a `workspace-section` (repo picker, styled as full-width
git-branch-tag rows since only one repo is active at a time) vs `skills-section`
(mode toggles, stay as compact chips) vs history (redone as git-log-style
entries: session's short hash prominent above the title, using the
click-to-copy session ID feature). Welcome/empty state redone to teach the
interface (eyebrow + clickable example prompts) instead of purely decorative
italic text.
